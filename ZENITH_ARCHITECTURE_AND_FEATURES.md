# Zenith: Global Satellite & Orbital Anomaly Tracking Dashboard

**Zenith** is a highly interactive, real-time web application built for tracking satellites, orbital anomalies, and space debris across the Earth and our Solar System. Inspired by the visual aesthetics of NASA's "Eyes on the Solar System" and modern military-grade radar HUDs, Zenith acts as a live 3D orbital tracking matrix.

## 🌟 Core Features

### 1. Real-Time Radar Dashboard (`/radar`)
The primary interface provides a massive, high-fidelity radar display tracking thousands of real satellites orbiting Earth.
- **Web Worker Multithreading:** Parses and propagates SGP4 satellite calculations (from active TLE text data) entirely in a background web worker. This guarantees 60FPS UI performance even when tracking over 8,000 objects.
- **Geospatial Location Picker:** A full interactive 3D globe (powered by `react-globe.gl` and `Three.js`) allowing the user to select specific coordinates on Earth to center the radar array.
- **Dynamic Orbital Filtering:** Calculates precise Azimuth, Elevation, and Range relative to the user's selected coordinates. Automatically filters objects that are below the horizon.
- **3D Object Profile Analysis:** Clicking an anomaly on the radar opens an interactive 3D profile of the object (distinguishing nominal payloads from decaying debris) alongside core kinematics.
- **Live Orbital Feeds:** Integrates 24/7 high-definition streams directly from space (e.g., ISS HDEV, SpaceX Starlink) to monitor physical assets in real-time.

### 2. Live 3D Orbital Tracking Matrix (`/solar-system`)
A full-screen, cinematic 3D environment inspired directly by **NASA Eyes**.
- **Procedural Solar System Engine:** Renders the Earth with atmosphere, the Moon, Mars, Jupiter, Saturn (with rings), Uranus, and Neptune to scale within the local viewing matrix.
- **NASA Eyes True Replication:** Features a highly detailed, cinematic UI complete with interactive Featured Targets (e.g., ISS, Hubble, SMAP, Starlink).
- **Cinematic Flybys:** The camera smoothly tracks selected objects in 3D space, showing real-time velocity (km/s) and orbital paths mapped perfectly around the Earth.
- **Procedural 3D Models:** Renders high-fidelity procedural 3D models for specific high-value satellites (like the SMAP satellite with its massive spinning radar dish and boom arm).
- **Temporal Shift Slider:** Allows the user to move forward and backward in time, instantly recalculating all orbits and planetary positions to visualize past or future orbital alignments.

## 🚀 Technical Architecture

### Tech Stack
- **Framework:** Next.js 14 App Router
- **Language:** TypeScript
- **Styling:** TailwindCSS
- **State Management:** Zustand (for global cross-component state, theme colors, and temporal shifting)
- **3D Rendering Engine:** Three.js, `@react-three/fiber`, `@react-three/drei`
- **Geospatial & Globe rendering:** `react-globe.gl`, `three-globe`
- **Orbital Mechanics:** `satellite.js` (for SGP4/SDP4 propagation of TLEs)

### System Design Highlights
1. **The Web Worker Pipeline:**
   - The main thread fetches `active.txt` containing thousands of TLEs (Two-Line Elements).
   - The TLEs are passed to a background Web Worker (`orbitalWorker.ts`).
   - Every 1,000ms, the worker propagates the entire array to the current time, calculating the absolute Earth-Centered Earth-Fixed (ECEF) coordinates and geodetic relative positions (Azimuth/Elevation).
   - Only visible objects are passed back to the main thread, keeping React's DOM rendering lean and performant.

2. **Framer Motion vs Three.js Hybrid Rendering:**
   - The application masterfully blends standard DOM elements (animated with Framer Motion for glassy, military-HUD aesthetics) with raw WebGL elements in `<Canvas>` blocks.

3. **Customization & Themes:**
   - A global `useRadarStore` handles the active CSS color variable definitions. When a user changes the color palette, the entire application (including Three.js 3D meshes) reacts dynamically to the new hexadecimal mappings.
